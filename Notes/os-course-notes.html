Lesson 1: Introduction to Computer System and Operating System

1.1 Computer System Components

Hardware

Provides basic computing resources (CPU, memory, I/O devices)

Operating System

Controls and coordinates hardware usage among applications

Application Programs

Define how system resources solve computing problems

Users

People, machines, other computers

1.2 Von Neumann Architecture

First published by John von Neumann in 1945

Design includes a Control Unit (CU), Arithmetic and Logic Unit (ALU), Memory Unit, Registers, and Inputs/Outputs.

üèóÔ∏è Von Neumann Architecture Components

Based on the stored-program computer concept, where instruction and program data are stored in the same memory.

1.2.1 Central Processing Unit (CPU)

Electronic circuit responsible for executing computer program instructions, also called microprocessor or processor. Contains the ALU, CU, and various registers.

1.2.1.1 Registers

High-speed storage areas in the CPU where all data must be stored before processing:

  • MAR (Memory Address Register): Holds memory location of data to be accessed
  • MDR (Memory Data Register): Holds data being transferred to or from memory
  • AC (Accumulator): Stores intermediate arithmetic and logic results
  • PC (Program Counter): Contains the address of the next instruction to be executed
  • CIR (Current Instruction Register): Contains the current instruction during processing
1.2.1.2 Arithmetic and Logic Unit (ALU)

Performs arithmetic (add, subtract) and logic (AND, OR, NOT) operations.

1.2.1.3 Control Unit (CU)

Controls the operation of the computer's ALU, memory, and I/O devices, and provides timing/control signals.

1.2.3 Buses

Means by which data is transmitted between computer parts, connecting major internal components to CPU and memory. Comprised of:

  • Address Bus: Carries addresses of data between processor and memory
  • Data Bus: Carries data between processor, memory, and I/O devices
  • Control Bus: Carries control signals/commands from CPU to control and coordinate activities

1.2.4 Memory Unit

Consists of RAM (Random Access Memory) and ROM (Read Only Memory), also known as primary or main memory. Fast and directly accessible by the CPU, unlike hard drives (secondary memory).

1.2.5 Input Devices

Examples: Keyboard, Microphone, Scanner, Mouse, Trackball, Touchpad, Barcode/QR Code readers, Digital Camera

1.2.6 Output Devices

Examples: Screen (Monitor/Console), Data Projectors, Speaker and Headphones, Printer, Plotter, Cutter

1.3 Computer Operating System

A system is components integrated to perform a desired task. A computer operating system consists of integrated components for operating the computer system (hardware & software).

Main Objectives of an OS

Efficiency

Responsiveness, fluidity

Ease of Usability

User-friendly interface

Abstraction

Extend to new devices and software

OS Categories - Usage Types

  • Batch: Sequential, repeated execution without human intervention
  • Time Sharing: Multiple users/applications access common hardware
  • Parallel Distributed: Single OS coordinates physically distributed hardware
  • Network: Systems connected over network protocols
  • Real Time: Fine-grained time precision in execution

Lesson 2: Processes and Scheduling

2.2 Program and Process Concept

Program

Static set of capabilities, processed code built for the OS

Process

An executing program; OS assigns process ID and tracking metrics

2.3 Process Contents

  • Text section: Program instructions
  • Program Counter (PC): Next instruction address
  • Stack Section: Local variables, return addresses, method parameters
  • Heap Section: Dynamic allocation (at run-time)
  • Data Section: Global and Static Variables

2.4 Process States

New
Ready
Running
Waiting
Terminated
  • New: The process is being created
  • Running: Instructions are being executed
  • Waiting: Process is waiting for some event to occur
  • Ready: Process is waiting to be assigned to a processor
  • Terminated: Process has finished execution

2.5 Process Control Block (PCB)

The PCB contains essential information about each process:

  • Unique identifier (process ID)
  • Process state
  • Pointer (process hierarchy)
  • Priority level
  • Program counter
  • CPU registers
  • Memory management info
  • I/O status information

2.7 Scheduling

Ready Queue

Processes ready to be scheduled

Waiting Queue

Processes blocked by I/O or devices

Job Queue

All processes in the system

Scheduling Algorithms

  • First Come, First Serve (FCFS)
  • Shortest Job First (SJF)
  • Shortest Remaining Time First (SRTF)
  • Round-Robin
  • Static/Dynamic Priority

2.8 Thread Concepts

Thread: A lightweight process. A process can create one or more threads internally for execution on the processor.

Examples: Chatting programs (sending/receiving independently), strategic games (multiple simultaneous actions)

Lesson 3: Memory Management

3.1 Need of Memory Management

In systems with multiple programs running, many processes may reside in memory, each with specific memory needs:

  • Executable code (instructions)
  • Data part of the executable
  • Runtime memory requests from stack/heap
  • OS and kernel components
  • Device-specific memory portions

3.2 Address Binding

Symbolic Addresses

References to variables, instructions, and source code

Logical Address

Translated symbolic addresses after compilation

Physical Address

Actual memory location in hardware

MMU

Memory Management Unit translates addresses

3.3 Inter-process Communication (IPC)

4.7 Synchronization

üîí Mutex

Mutual exclusion - only one process can hold the key

üö¶ Semaphore

4.8 Deadlocks

Four Conditions for Deadlock:

  1. Mutual Exclusion: Resources not shareable
  2. Hold and Wait: Process holds resources while waiting for others
  3. No Preemption: Resources cannot be forcefully taken
  4. Circular Wait: Processes waiting for each other in circular fashion

Lesson 7: Virtualization and User Interface

7.1 Virtualization

A feature provided by OSs and modern hardware that virtualizes hardware, making each calling environment believe it has dedicated access.

Advantages of Virtualization

  • Run OSs when physical hardware is unavailable
  • Easier to create new machines, backup machines
  • Software testing using "clean" installs
  • Emulate more machines than physically available
  • Debug problems (suspend and resume)
  • Easy migration of virtual machines
  • Run legacy systems
