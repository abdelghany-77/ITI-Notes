‚öõÔ∏è Module 1: React Fundamentals

Understanding React's core philosophy and architecture

üìö What is React?

React is a JavaScript library for building user interfaces, developed and maintained by Facebook (Meta). It uses a component-based architecture and declarative programming paradigm to create interactive UIs efficiently.

Key Features:

  • Declarative: Describe what the UI should look like, React handles the how
  • Component-Based: Build encapsulated components that manage their own state
  • Learn Once, Write Anywhere: React Native for mobile, React for web
  • Virtual DOM: Efficient rendering through DOM diffing algorithm
  • Unidirectional Data Flow: Predictable state management

üéØ Virtual DOM Explained

The Virtual DOM is a lightweight JavaScript representation of the actual DOM. React uses it to optimize rendering:

  • Step 1: When state changes, React creates a new Virtual DOM tree
  • Step 2: React compares (diffs) the new Virtual DOM with the previous one
  • Step 3: React calculates the minimum set of changes needed
  • Step 4: React updates only the changed parts in the real DOM

üí° Performance Benefit: Direct DOM manipulation is slow. By batching updates and minimizing DOM operations, React achieves better performance than traditional approaches.

üß© Module 2: Components & JSX

Building blocks of React applications

üéØ What are Components?

Components are independent, reusable pieces of UI. They accept inputs (props) and return React elements describing what should appear on screen.

Types of Components:

  • Function Components: Modern, simpler, use hooks
  • Class Components: Legacy, more boilerplate, have lifecycle methods

üìù Function Components

// Basic Function Component
function Welcome(props) {
  return <h1>Hello, {props.name}!</h1>;
}

// Arrow Function Component
const Welcome = (props) => {
  return <h1>Hello, {props.name}!</h1>;
};

// Implicit Return (for simple components)
const Welcome = ({ name }) => <h1>Hello, {name}!</h1>;

// Usage
<Welcome name="John" />

üí° Best Practice: Use function components with hooks for all new code. They're simpler, more readable, and the React team recommends them.

üèõÔ∏è Class Components (Legacy)

import React, { Component } from 'react';

class Welcome extends Component {
  constructor(props) {
    super(props);
    this.state = {
      count: 0
    };
  }

  componentDidMount() {
    console.log('Component mounted');
  }

  componentDidUpdate(prevProps, prevState) {
    console.log('Component updated');
  }

  componentWillUnmount() {
    console.log('Component will unmount');
  }

  render() {
    return (
      <div>
        <h1>Hello, {this.props.name}!</h1>
        <p>Count: {this.state.count}</p>
        <button onClick={() => this.setState({ count: this.state.count + 1 })}>
          Increment
        </button>
      </div>
    );
  }
}

export default Welcome;

‚ö†Ô∏è Note: While class components still work, function components with hooks are the modern standard. Learn classes only for maintaining legacy code.

‚ú® JSX Syntax

JSX (JavaScript XML) is a syntax extension that looks like HTML but gets compiled to JavaScript. It allows you to write HTML structures in the same file as JavaScript code. 

// JSX Example
const element = <h1 className="greeting">Hello, World!</h1>;

// Compiles to:
const element = React.createElement(
  'h1',
  { className: 'greeting' },
  'Hello, World!'
);

// JSX Rules:
// 1. Must return single parent element
const component = (
  <div>
    <h1>Title</h1>
    <p>Paragraph</p>
  </div>
);

// 2. Use React.Fragment or <></> for multiple elements
const component = (
  <>
    <h1>Title</h1>
    <p>Paragraph</p>
  </>
);

// 3. JavaScript expressions in curly braces
const name = "John";
const element = <h1>Hello, {name}!</h1>;

// 4. Use className instead of class
const element = <div className="container">Content</div>;

// 5. Use htmlFor instead of for
const label = <label htmlFor="name">Name:</label>;

// 6. Self-closing tags must have /
const image = <img src="photo.jpg" alt="Photo" />;
const input = <input type="text" />;

‚ö†Ô∏è Common JSX Mistakes

  • Using class instead of className: Will cause warnings
  • Forgetting to close tags: All tags must be closed or self-closing
  • Missing keys in lists: Required for React's reconciliation
  • Direct if statements: Use ternary or && operators instead
  • Modifying props: Props are read-only, never mutate them

üé® Module 3: Props & State

Managing data in React components

üì¶ Props (Properties)

Props are arguments passed into React components. They are read-only and flow from parent to child (unidirectional data flow). 

// Passing Props
function App() {
  return (
    <div>
      <Welcome name="Alice" age={25} isActive={true} />
      <Welcome name="Bob" age={30} isActive={false} />
    </div>
  );
}

// Receiving Props
function Welcome(props) {
  return (
    <div>
      <h1>Hello, {props.name}!</h1>
      <p>Age: {props.age}</p>
      <p>Status: {props.isActive ? 'Active' : 'Inactive'}</p>
    </div>
  );
}

// Destructuring Props (Preferred)
function Welcome({ name, age, isActive }) {
  return (
    <div>
      <h1>Hello, {name}!</h1>
      <p>Age: {age}</p>
      <p>Status: {isActive ? 'Active' : 'Inactive'}</p>
    </div>
  );
}

// Default Props
function Welcome({ name = "Guest", age = 0 }) {
  return <h1>Hello, {name}, Age: {age}</h1>;
}

// Prop Types (Type Checking)
import PropTypes from 'prop-types';

Welcome.propTypes = {
  name: PropTypes.string.isRequired,
  age: PropTypes.number,
  isActive: PropTypes.bool
};

Welcome.defaultProps = {
  age: 18,
  isActive: false
};

üîÑ State Management

State is data that changes over time within a component. Unlike props, state is managed within the component and can be updated. 

import { useState } from 'react';

// Basic State Example
function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>Increment</button>
      <button onClick={() => setCount(count - 1)}>Decrement</button>
      <button onClick={() => setCount(0)}>Reset</button>
    </div>
  );
}

// Multiple State Variables
function Form() {
  const [name, setName] = useState('');
  const [email, setEmail] = useState('');
  const [age, setAge] = useState(0);

  return (
    <form>
      <input 
        value={name} 
        onChange={(e) => setName(e.target.value)} 
        placeholder="Name"
      />
      <input 
        value={email} 
        onChange={(e) => setEmail(e.target.value)} 
        placeholder="Email"
      />
      <input 
        type="number"
        value={age} 
        onChange={(e) => setAge(Number(e.target.value))} 
      />
    </form>
  );
}

// Object State
function UserProfile() {
  const [user, setUser] = useState({
    name: '',
    email: '',
    age: 0
  });

  const updateUser = (field, value) => {
    setUser(prevUser => ({
      ...prevUser,
      [field]: value
    }));
  };

  return (
    <div>
      <input 
        value={user.name}
        onChange={(e) => updateUser('name', e.target.value)}
      />
      <input 
        value={user.email}
        onChange={(e) => updateUser('email', e.target.value)}
      />
    </div>
  );
}

// Array State
function TodoList() {
  const [todos, setTodos] = useState([]);
  const [input, setInput] = useState('');

  const addTodo = () => {
    setTodos([...todos, { id: Date.now(), text: input }]);
    setInput('');
  };

  const removeTodo = (id) => {
    setTodos(todos.filter(todo => todo.id !== id));
  };

  return (
    <div>
      <input 
        value={input}
        onChange={(e) => setInput(e.target.value)}
      />
      <button onClick={addTodo}>Add</button>
      <ul>
        {todos.map(todo => (
          <li key={todo.id}>
            {todo.text}
            <button onClick={() => removeTodo(todo.id)}>Delete</button>
          </li>
        ))}
      </ul>
    </div>
  );
}

‚öñÔ∏è Props vs State

Aspect Props State
Definition Data passed from parent Data managed within component
Mutability Read-only (immutable) Can be changed with setState
Ownership Owned by parent Owned by component
Update Trigger Parent re-renders setState call
Use Case Component configuration Interactive/dynamic data

üéØ Best Practices

  • Immutability: Never mutate state directly, always use setState
  • Functional Updates: Use function form when new state depends on previous
  • Lift State Up: Move shared state to closest common ancestor
  • Single Source of Truth: Don't duplicate state across components
  • Derived State: Calculate values from existing state instead of storing them
