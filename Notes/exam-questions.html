Q47

b. Ensuring that the system is acceptable to all users
c. Testing the system with other systems

Correct Answer: d. Testing for a business perspective

Acceptance testing validates that the system meets business requirements and is ready for deployment.

Q48

b. Should be able to understand the source code
c. Is highly motivated to find faults
d. is creative to find the system's weaknesses

Correct Answer: b. Should be able to understand the source code

Black box testing doesn't require knowledge of internal code structure; it focuses on functionality.

Q49

b. Performed by customers at their software developer's site
c. Performed by an independent test team
d. Useful to test bespoke software
e. Performed as early as possible in the lifecycle

Correct Answer: a. Performed by customers at their own site

Beta testing is conducted by actual users in their real environment before final release.

Q50

a. Checking that we are building the right system
b. Checking that we are building the system right
c. Performed by an independent test team
d. Making sure that it is what the user really wants

Correct Answer: b. Checking that we are building the system right

Verification ensures the system is built according to specifications. Validation checks if it's the right system.

Q51
The difference between re-testing and regression testing is:
a. Re-testing is running a test again regression testing looks for unexpected side effects

Correct Answer: a. Re-testing is running a test again regression testing looks for unexpected side effects

Re-testing verifies fixes work; regression testing ensures changes don't break existing functionality.

Q52
Before launching a software which testing is to be done in-house?

b. Gamma
c. Alpha
d. None of the above

Correct Answer: c. Alpha

Alpha testing is performed in-house by internal teams before external beta testing.

Q53
Software testing activities should start

Q54

c. Bad luck

Q55

Q56

White box testing examines the internal structure, code paths, and logic of the software.

Q57

Q58

Compilation testing is not a recognized testing level. The main levels are Unit, Integration, System, and Acceptance testing.

Q59

a. Unit Acceptance Testing
b. User Acceptance Testing
c. Universal Access Testing

Correct Answer: b. User Acceptance Testing

UAT is User Acceptance Testing, where end users test the system to ensure it meets their requirements.

Q60

Black box testing focuses on input-output behavior without knowledge of internal code structure or implementation details.

What is the difference between gzip and tar?
gzip archives, tar compresses
gzip is used to compress a single file; tar creates multi-file archives
gzip creates archives with metadata; tar doesn't

Explanation:

gzip compresses individual files, while tar creates archives of multiple files. They're often used together (tar.gz).
