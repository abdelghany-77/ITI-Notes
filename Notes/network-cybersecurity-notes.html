Session 1: Network Essentials

Computer Network Definition

üåê What is a Computer Network?

A collection of computers and other devices or peripherals connected through connecting media to perform tasks like sharing resources.

Resources include: File sharing, Device sharing, Software sharing with multi-user licenses, Voice and Video calls, Shared Internet Access.

Network Elements

üñ•Ô∏è Hardware

Devices like Computers, Printers, Phones, Routers, Switches

üì° Medium

Wired, Wireless, Satellites

üíª Software

Messages: Information traveling over medium (Mails, WhatsApp)

Protocols: Communication rules (HTTP, HTTPS, FTP, RDP)

Network Basic Terminologies

üîå NIC (Network Interface Card)

Hardware enabling device to access network. Can be internal or external (Wireless/USB based)

üè∑Ô∏è MAC Address

Physical Address, unique worldwide, burned on the NIC card

üåê IP Address

Logical address, identifies each device on an IP network layer

üìã Protocols

Communication rules that all entities must agree on

üîÑ Hub

Allows different nodes to communicate but slows the network

üì∂ Repeater

Regenerates signal before it becomes weak or corrupted

üì° Access Point (AP)

Allows Wi-Fi devices to connect to wired network

üîÄ Switch

Allows nodes to communicate without slowing each other

üåç Router

Allows different networks to communicate with each other

Network Classifications

üìä According to Covered Area

LAN (Local Area Networks): Small geographical area (school, campus, office). High communication speed, allows file/service sharing, under administrative control.

WAN (Wide Area Networks): Large geographical area (country). Can contain multiple LANs/MANs. Very low speed, under ISP control. The Internet is a WAN example.

üîó According to Network Model

Peer-to-Peer: No dedicated resources, easy to work with, all nodes equal in resource usage.

Client/Server: Some nodes (servers) dedicated to providing services to other nodes (clients). Servers are more powerful (Mail, Web, File, Print Servers).

The Internet

üåç Global Mesh of Interconnected Networks

No one owns the Internet; many organizations, ISPs, companies, and governments own pieces of its infrastructure.

Key organizations: ISOC (Internet Society), IETF (Internet Engineering Task Forum), ICANN (Internet Corporation for Assigned Names and Numbers)

Application Layer Protocols

üåê HTTP

Hyper Text Transfer Protocol - Supports delivery of web pages to client

üîç DNS

Domain Name Servers - Translates human-readable names into IP addresses

üìÅ FTP

File Transfer Protocol - Provides reliable data transfer between hosts

üîê SSH

Secure Shell - Port 22, very secure, encrypted remote access

üìß SMTP

Simple Mail Transfer Protocol - Sends messages between Mail Servers

üì¨ POP3/IMAP4

Email retrieval protocols - POP3 downloads, IMAP4 retrieves headers

Transport Layer Protocols

üîÑ TCP (Transmission Control Protocol)

Features: Reliable, Connection-oriented, Full duplex, Error control, Data-recovery

Used for: Email, File sharing, Downloading

‚ö° UDP (User Datagram Protocol)

Features: Unreliable, Connectionless, Full duplex, Limited error checking

Used for: Voice Streaming, Video Streaming

Session 2: Cyber Security Essentials

Cyber Security Definition

üîí What is Cyber Security?

Protection of systems, networks, programs, devices, and data from cyber attacks. Security is a shared responsibility.

Security Goals (CIA Triad)

ü§ê Confidentiality

Ensuring information is not revealed to unauthorized persons

‚úÖ Integrity

Ensuring consistency of data and ability to detect modifications

üåê Availability

Ensuring legitimate users are not denied access to information

Risks & Threats

‚ö†Ô∏è Risk Formula

Risk = Threat √ó Vulnerabilities

Vulnerability: Degree of weakness in every network and device

Threat: A person, thing, event, or idea posing danger to an asset

Attacker Methodology

üîç Reconnaissance

Gathering information about target before attack

üì° Scanning

Scanning network for open ports, OS, applications

üéØ Gaining Access

True attack phase where hacker exploits the system

üîí Maintaining Access

üßπ Covering Tracks

Attack Methods

üé≠ Social Engineering Attacks

  • Phishing: Fake web pages to steal credentials
  • Session Hijacking: Taking over sessions between parties
  • Insider Attack: Attacks from within the organization
  • Spoof Attack: Modifying source addresses to appear from someone else
  • Password Attack: Dictionary, Brute-force, Hybrid attacks

üìß Spam

üëÅÔ∏è Spyware

Session 3: Distributed Systems

Distributed Systems Overview

‚òÅÔ∏è What are Distributed Systems?

A group of computers working together appearing as a single computer to the end-user. Collection of independent components on different machines sharing messages to achieve common goals.

Centralized vs. Distributed

üè¢ Centralized System

Pros: Simpler, easier to understand, faster for single user

Cons: Single point of failure, limited scalability

üåê Distributed System

Pros: More robust, scalable, fault-tolerant

Cons: More complex to design and manage

Virtualization

üíª Virtualization Technology

Technology to run multiple same or different operating systems, completely isolated, simultaneously on the same machine.

Benefits: Hardware consolidation, Redundancy, Migration, Centralized management

Cloud Computing

‚òÅÔ∏è Cloud Computing Definition

A pool of resources that can be rapidly provisioned in an automated, on-demand manner.

Value: Economies of scale, Elasticity, Cost benefits, Easy access regardless of location

üè¢ On-Premises

üõ†Ô∏è IaaS

‚öôÔ∏è PaaS

üì± SaaS

Cloud Deployment Models

üåç Public Cloud

Open to general public, owned by third party

üè¢ Private Cloud

üîÑ Hybrid Cloud

üë• Community Cloud
