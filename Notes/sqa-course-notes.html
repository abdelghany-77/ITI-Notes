  • Faulty requirements definition
  • Client-developer communication failures

  • Logical design errors
  • Coding errors
  • Non-compliance with documentation and coding instructions
  • Shortcomings of the testing process
  • Documentation errors

5. Bug Life Cycle

States of a Bug

NEW
OPEN
ASSIGN
REJECTED
DEFERRED
TEST
REOPENED
VERIFIED
CLOSED

üîÑ State Descriptions:

  • NEW: Newly reported bug
  • OPEN: Bug is open for review/assignment
  • ASSIGN: Bug is assigned to a developer
  • REJECTED: Bug is rejected (not a bug, duplicate)
  • DEFERRED: Bug is deferred for later fix
  • TEST: Bug is fixed and ready for testing
  • REOPENED: Bug is reopened if fix is not satisfactory
  • VERIFIED: Bug fix is verified by tester
  • CLOSED: Bug is closed after verification
