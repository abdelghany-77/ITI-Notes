🔓 Open Source Principles

🆓 Free Software

Free ≠ No Money

  • No cost to use, download, or distribute
  • Freedom from licensing fees
  • No vendor lock-in
  • Community-driven development

🔍 Freedom

Four Essential Freedoms:

  • View: Access to source code
  • Modify: Change the software
  • Distribute: Share with others
  • Improve: Contribute back to community

🧠 Understanding Linux Architecture

🔑 Key Concept:

Kernel ≠ Operating System

  • Kernel: Heart of the OS - manages hardware
  • Distribution: Complete OS = Kernel + GUI + Package Manager + Libraries
# Check your kernel version
uname -r
5.15.0-72-generic
# View system information
uname -a
Linux ubuntu 5.15.0-72-generic #79-Ubuntu SMP x86_64 GNU/Linux
# Check distribution info
lsb_release -a
cat /etc/os-release

🛣️ Understanding Paths

📐 Path Types:

Relative Path:
  • Depends on current working directory
  • Starts with ./, ../, or filename
  • Examples: ./file.txt, ../parent/
Absolute Path:
  • Depends on root directory /
  • Starts with / or ~
  • Examples: /home/user, ~/Documents

🗺️ File System Structure Example

🌳 Directory Tree:

/ ├── home/ │ ├── ali/ │ │ └── test/ │ │ └── f.txt │ └── mina/ │ └── test2/ │ └── f2.txt └── etc/ └── passwd

🧭 Path Navigation Examples

# Current location: /home/ali/test
pwd
/home/ali/test
# Relative path examples:
cat f.txt # Current directory file
cat ./f.txt # Explicit current directory
cat ../../mina/test2/f2.txt # Navigate to other user's file
cat ../../../etc/passwd # System file via relative path
# Absolute path examples:
cat /home/ali/test/f.txt # Direct path to current file
cat /home/mina/test2/f2.txt # Direct path to other file
cat /etc/passwd # Direct path to system file

👑 Root Directory Concepts

🔑 Understanding Root:
  • / - Root directory (file system start point)
  • root - Root user (administrator account)
  • /root - Root user's home directory

🏠 Accessing Root Directories:

# Go to root directory (file system root)

# Go to root user's home directory
# Method 1: Login as root first

# Method 2: Direct access (if permissions allow)

✅ Valid vs Invalid Path Operations

✅ Valid Operations

# Directory navigation

# File access

❌ Invalid Operations

# Can't add trailing slash to files
cat /etc/passwd/ # ERROR

# Man can't take file paths
man /etc/passwd # ERROR

📁 4. File & Directory Operations

✨ Creating Files and Directories

📄 Creating Files

# Single file
touch filename.txt

# Multiple files
touch file1 file2 file3

# Using brace expansion
touch ~/Music/{ahmed,mina}

# Sequential numbering
touch ~/Music/file{1..100}.txt
touch ~/Music/file{a..z}.txt

# Multiple locations
touch ~/{Music,Videos}/file

📁 Creating Directories

# Single directory
mkdir dirname

# Multiple directories
mkdir dir1 dir2 dir3

# Using brace expansion
mkdir ~/Music/{dir1,dir2}

# Create parent directories
mkdir -p ~/Music/dir1/dir12

# Complex nested structure
mkdir -p ~/Music/dir2/{dir11,dir12}

🔄 File and Directory Rules

📝 Creation Rules:

# You can create a file even if directory exists with same name
mkdir test
touch test # No error - file and directory coexist

# You cannot create directory if file exists with same name
touch test2
mkdir test2 # Error: File exists

# Use -p flag to create parent directories
mkdir ~/Music/dir1/dir12 # Error: No such file or directory
mkdir -p ~/Music/dir1/dir12 # Success: Creates all parents

📋 Copying Files and Directories

📐 Copy Command Logic:

  • If destination is existing folder: Copy file with same name
  • If destination doesn't exist: Copy file with new name
  • If destination is existing file: Overwrite (use -i for confirmation)
# Basic file copying
cp /etc/passwd ~/Music/ # Copy with same name
cp /etc/passwd ~/Music/mypasswd # Copy and rename

# Interactive copying (asks before overwrite)
cp -i /etc/passwd ~/Music/mypasswd
# cp: overwrite '/home/mina/Music/mypasswd'? y

# Copy multiple files (destination must be existing directory)
cp /etc/{passwd,group} ~/Music/
cp ~/git/file{1..100} ~/Music/

# Copy directories recursively
cp -r ~/Videos/ ~/Music/
cp -ir ~/Videos/ ~/Music/ # Interactive recursive copy

🚚 Moving and Renaming

📦 Moving Files/Directories

# Move files to directory
mv ~/Music/group ~/Videos/
mv ~/Music/test ~/Videos/

# Move multiple files
mv ~/Music/{passwd,group} ~/Videos/
mv ~/Music/{test,test2} ~/Videos/

✏️ Renaming Files/Directories

# Rename files
mv ~/Music/dir1 ~/Music/dir11
mv ~/Music/{dir1,dir11} # Shorthand

# Rename files
mv ~/Music/file1 ~/Music/file11
mv ~/Music/{file1,file11} # Shorthand

🗑️ Removing Files and Directories

⚠️ Deletion Warning: Linux doesn't have a recycle bin! Deleted files are permanently removed.

🗂️ Remove Files

# Remove single file
rm filename

# Interactive removal
rm -i ~/Music/group
# rm: remove regular file '/home/mina/Music/group'? y

# Remove multiple files
rm ~/Music/file{1..100}.txt
rm -i ~/Music/group ~/Videos/file

📁 Remove Directories

# Remove empty directory
rmdir foldername

# Remove directory with contents
rm -r ~/Music/Videos/
rm -ir ~/Music/Videos/ # Interactive

# Remove parent directories if empty
rmdir -p /home/mina/Music/Videos/

🧹 Bulk Operations

🗂️ Managing Multiple Items:

# Create multiple directories
mkdir ~/Music/{dir1,dir2}

# Create multiple files
touch ~/Music/file{1..100}

# Delete all contents of directory (keep directory)
rm -r ~/Music/*

# Complex path expansions
# These create files in both directories:
# ~/Music/git/test/file and ~/Music/git/test1/file
touch ~/Music/git/{test,test1}/file

🔧 Advanced Copy Operations

# Copy with specific conditions
cp -ir ~/Videos/ ~/Music/myvideos # Rename during copy

# Copy to multiple destinations
cp -ir ~/Videos/ ~/Desktop ~/Music # Copy to both locations

# Error handling
cp -ir ~/Videos/ ~/Music/group
# cp: cannot overwrite non-directory
# (This happens when trying to copy directory over existing file)
🚨 Dangerous Commands - Use with Caution:
  • rm -rf / - Could delete entire system (usually protected)
  • rm -r * - Deletes everything in current directory
  • mv important_folder /tmp/ - Could lose important data

💡 Safety Tip: Always use -i flag for interactive confirmation!

👁️ 5. Viewing File Contents

📖 Basic File Viewing

📄 Display Entire File

cat filename
cat -n filename # with line numbers

Shows all content at once

📖 Page-by-Page Viewing

more filename
less filename

Navigate through large files

⚠️ Cat Without Arguments

cat # Holds terminal
# Press Ctrl+C to exit

Without arguments, cat waits for input

📊 Selective File Viewing

Command Description Options Examples
head First 10 lines -n number, -number head -5 /etc/passwd
tail Last 10 lines -n number, -number tail -3 /etc/passwd
head -n2 First 2 lines Custom count head -2 /etc/passwd
tail -n1 Last 1 line Custom count tail -1 /etc/passwd

🔀 Pipelines - Chaining Commands

🧩 Pipeline Concept:

Command1 | Command2 - Output of Command1 becomes input of Command2

# Basic pipeline examples
ps -ef | less # View processes page by page
sort /etc/passwd | less # Sort users and view
cat /etc/passwd | sort | less # Chain multiple commands

# File processing with line numbers
cat -n /etc/passwd | head # Number lines, show first 10
cat -n /etc/passwd | tail -1 # Show last line with number
sort /etc/passwd | cat -n | head # Sort, number, then show first 10

🎯 Extracting Specific Line Ranges

📊 Advanced Line Selection (Lines 5-11):

# Extract lines 5 to 11 from /etc/passwd
cat -n /etc/passwd | head -11 | tail -7

# How it works:
# 1. cat -n /etc/passwd → Number all lines
# 2. | head -11 → Take first 11 lines (1-11)
# 3. | tail -7 → Take last 7 of those (5-11)

# Expected output:
# 5 sync:x:4:65534:sync:/bin:/bin/sync
# 6 games:x:5:60:games:/usr/games:/usr/sbin/nologin
# 7 man:x:6:12:man:/var/cache/man:/usr/sbin/nologin
# 8 lp:x:7:7:lp:/var/spool/lpd:/usr/sbin/nologin
# 9 mail:x:8:8:mail:/var/mail:/usr/sbin/nologin
# 10 news:x:9:9:news:/var/spool/news:/usr/sbin/nologin
# 11 uucp:x:10:10:uucp:/var/spool/uucp:/usr/sbin/nologin

📋 Directory Listing

# Basic directory listing
ls # List working directory

# List with file type indicators

# @ = symlink, / = directory, * = executable

🔍 File Type Identification

Symbol File Type Example
@ Symbolic link bin@ → usr/bin
/ Directory home/
* Executable file install-script.sh*
(none) Regular file document.txt

🔍 Filtering Directory Listings

# Filter by file type
ls -F / | grep '/' # Show only directories
ls -F / | grep '*' # Show only executable files
# install-script.sh*

# Filter by pattern
ls -F / | grep 'in'
# bin@
# install-script.sh*
# sbin@

# Filter home directory
ls -F /home/mina | grep '/' # Show only subdirectories
ls -F ~ | grep '/' # Same as above

👥 6. User & Group Management

🏷️ User Types in Linux

👑 Root User

UID: 0

System administrator with unlimited privileges. Can perform any operation on the system.

Switch to root:
sudo -i - Interactive login
sudo -s - Non-login shell
su - - Full root environment
su root - Basic switch

👤 Normal Users

UID: 1000+

Regular users with limited privileges. Each user has their own home directory and settings.

User operations:
whoami - Current user
id username - User details
groups username - User groups
finger username - User info

⚙️ Service Users

UID: 1-999

System accounts for running services and daemons. Usually don't have login shells.

Common examples:
www-data - Web server
mysql - Database service
mail - Mail service
nobody - Minimal privileges

📋 User Information Files

👥 /etc/passwd - User Database

# View all users
cat /etc/passwd

# Format: user:x:uid:gid:comment:home:shell
# Example:
root:x:0:0:root:/root:/bin/bash
mina:x:1000:1000:Mina User:/home/mina:/bin/bash

Fields: Username, Password (x=encrypted), UID, GID, Comment, Home Directory, Shell

👥 /etc/group - Group Database

# View all groups
cat /etc/group

# Format: group:x:gid:members
# Example:
sudo:x:27:mina,alice
developers:x:1001:john,jane,bob

Fields: Group name, Password, GID, Group members

🔐 Password and Security Files

🛡️ Shadow Files (Root Access Required):

# User passwords (encrypted)
sudo cat /etc/shadow
# Format: user:encrypted_password:last_change:min:max:warn:inactive:expire

# Group passwords
sudo cat /etc/gshadow
# Format: group:encrypted_password:admins:members

➕ Creating Users

🔧 useradd Default Settings:

# View default settings
useradd -D

# Output:
GROUP=100
HOME=/home
INACTIVE=-1
EXPIRE=
SHELL=/bin/sh
SKEL=/etc/skel
CREATE_MAIL_SPOOL=no
# Basic user creation
sudo useradd username

# Create user with specific options
sudo useradd fatma -s /usr/bin/bash -m -d /home/fatma -c "Python Developer"

# Breakdown of options:
# -s /usr/bin/bash → Set shell
# -m → Create home directory
# -d /home/fatma → Specify home directory
# -c "Python Developer" → Add comment
# -u 1004 → Set specific UID
# -g groupname → Set primary group

# Set user password
sudo passwd fatma

# Verify user creation
id fatma
tail -1 /etc/passwd

🏠 User Home Directory Templates

📁 /etc/skel - Skeleton Directory:

Files and directories in /etc/skel are automatically copied to new user home directories.

# Customize new user defaults
sudo mkdir /etc/skel/{Documents,Music,Pictures}
sudo touch /etc/skel/.bashrc_custom

# Now all new users will get these directories

✏️ Modifying Users

Command Description Example
usermod -G group Set secondary groups (replaces) sudo usermod fatma -G sudo,developers
usermod -aG group Add to secondary groups sudo usermod fatma -aG docker
usermod -s shell Change user shell sudo usermod fatma -s /bin/zsh
usermod -d directory Change home directory sudo usermod fatma -d /opt/fatma
usermod -l newname Change username sudo usermod fatma -l fatima

👤 User Modification Examples:

# Add user to sudo group
sudo usermod fatma -aG sudo

# Check user groups
id fatma
# Output: uid=1004(fatma) gid=1004(fatma) groups=1004(fatma),27(sudo)

# Add to multiple groups
sudo usermod fatma -aG sudo,docker,developers

# Verify changes
groups fatma

🗑️ Removing Users

🗂️ Remove User (Keep Home)

sudo userdel username

Removes user account but keeps home directory and files

🗑️ Remove User (Delete All)

sudo userdel -r username

Removes user account, home directory, and all files

👥 Group Management

# Create group
sudo groupadd groupname
sudo groupadd developers -g 1500 # with specific GID

# Create system group (GID < 1000)
sudo groupadd -r system_group

# Modify group
sudo groupmod oldname -n newname # rename group
sudo groupmod groupname -g 1600 # change GID

# Delete group
sudo groupdel groupname

# Add user to group

🔍 Reading Permission Strings:

# Example: -rwxr-xr--
# │││││││││

📁 File vs Directory Permissions

Permission Files Directories
Read (r) View file contents List directory contents (ls)
Write (w) Modify file contents Create/delete files in directory
Execute (x) Run file as program Access directory (cd into it)
⚠️ Directory Permission Notes:
  • Need x permission to cd into a directory
  • Need r permission to list directory contents
  • Need w permission to create/delete files (even if you own the files)
  • Directory permissions override file permissions for access

🔄 Foreground vs Background Processes

🎯 Foreground Process

  • Runs in the terminal (holds terminal)
  • Blocks further input
  • Default execution mode
  • Example: gedit

🏃 Background Process

  • Runs without blocking terminal
  • Allows continued terminal use
  • Marked with & symbol
  • Example: sleep 100 &

tar (Tape Archive) combines multiple files/directories into a single archive file

Option Description Example
c Create archive tar cvf backup.tar /home
v Verbose (show files) Shows processing files
f Specify filename Must be followed by archive name
t List contents tar tvf backup.tar
x Extract archive tar xvf backup.tar

📦 Basic tar Operations

🗃️ Archive Management:

# 1. Create tar archive
tar cvf ~/Music/test3/etc.tar /etc

# 2. List archive contents
tar tvf ~/Music/test3/etc.tar

# 3. Extract to current directory
tar xvf ~/Music/test3/etc.tar

# 4. Extract to specific directory
tar xf ~/Music/test3/etc.tar -C ~/Music/test3/

# Size comparison example
du -sh /etc # 21M
tar cvf ~/Music/test3/etc.tar /etc
bzip2 ~/Music/test3/etc.tar
du -sh ~/Music/test3/etc.tar.bz2 # 1.9M

🚀 Combined tar + Compression

🗜️ tar + gzip (.tar.gz)

# Create compressed archive
tar czvf ~/Music/backup.tar.gz /etc

# Extract compressed archive
tar xzvf ~/Music/backup.tar.gz -C ~/target/

z flag: Use gzip compression

📦 tar + bzip2 (.tar.bz2)

# Create compressed archive
tar cjvf ~/Music/backup.tar.bz2 /etc

# Extract compressed archive
tar xjvf ~/Music/backup.tar.bz2 -C ~/target/

j flag: Use bzip2 compression

🎯 Compression Decision Tree

🤔 When to Use Which Tool:

🏃 Speed Priority

  • compress: Fastest, basic compression
  • gzip: Good balance of speed/compression
  • tar.gz: Fast archive + compression

💾 Size Priority

  • bzip2: Best compression ratio
  • tar.bz2: Best for archiving + max compression
  • zip: Cross-platform compatibility

⚠️ Important Notes:

  • Original Files: Most compression tools remove the original file by default
  • Directory Compression: Use tar first to create archive, then compress
  • Cross-platform: zip works on all operating systems
  • Unix Standard: tar.gz is the most common format on Linux/Unix
  • Extension Matters: Different tools create different file extensions
