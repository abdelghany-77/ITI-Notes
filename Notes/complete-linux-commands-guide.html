Day 1: Introduction to Linux, Basic Commands, and File Basics

1. Free/Open Source Software (FOSS) and Licenses

üÜì What is FOSS?

FOSS provides freedoms to:

  • View the source code used to compile programs
  • Make modifications
  • Distribute these modifications

Most FOSS is covered under a public license. The most common is the GNU General Public License (GPL).

üìú Open-source License

A type of license for computer software and other products that allows the source code, blueprint, or design to be used, modified, and/or shared under defined terms and conditions.

Examples: GPL, LGPL, Apache, Mozilla Public License, and BSD.

2. Linux History

üìö Evolution Timeline

  • Multics (1963-1969): Developed through collaboration of MIT, General Electric, and Bell Labs
  • Unix (1969): First version created in Bell Labs
  • Unix Flavors: IBM (AIX), Sun (Solaris), HP (HP/UX), Silicon Graphics (IRIX)
  • GNU Project (1983): Started by Richard Stallman with the goal of creating a free UNIX-like OS
  • Linux Kernel (1991): Created by Linus Torvalds
  • Integration (1992): Linux and GNU developers integrated GNU components with Linux

3. Why Linux & Red Hat

üêß Why Linux?

  • Linux is open source
  • Released under the GNU General Public License (GPL)
  • Secure and generally considered virus-free
  • Ideal for programmers
  • Strong community support
  • Dominates professional and server sectors

üé© Red Hat

  • Founded: March 26, 1993
  • First Release: Red Hat Linux in 1994
  • Market Presence: More than 90% of Fortune Global 500 companies use Red Hat
  • RHEL Benefits: Runs demanding applications better, scales well, reliable, and secure
  • CentOS: Community-driven, free alternative derived from RHEL source code

6. Basic Commands (CLI Fundamentals)

‚ö° Why Use CLI?

  • Consistent technical support across different GUIs
  • Speed & Performance (e.g., touch file{1..100})
  • Optimized for server-side environments
  • Security for remote connections (SSH)
Command Syntax:

Items separated by spaces
Separate multiple commands with semicolon (;)
Basic Commands:

echo $variable # Display variable value

üî§ Quoting Rules

  • Single quotes (' '): Strong quoting - ignore all special characters
  • Double quotes (" "): Weak quoting - expand variables but ignore some specials
Variables:

echo $HOME # Home directory
echo $USER # Current user

Command History:

  • /lib - Shared libraries

2. User Environment and Initialization Files

üåç Environment Creation

When a user logs in, an environment is created with variables determining how the user works. Initialization files are read in a specific order, and if a setting appears in multiple files, the last one read wins.

üìÇ Initialization File Order

  • Login Terminal: /etc/profile ‚Üí ~/.profile ‚Üí /etc/bash.bashrc
  • No Login Terminal (Subshell): /etc/bash.bashrc ‚Üí ~/.bashrc

üóÇÔ∏è Specific File Roles

File Scope When Used Purpose
/etc/profile System-wide Login shell Default settings for all users
/etc/bashrc System-wide Subshell Defaults for all users in subshells
~/.profile User-specific Login shell Personal settings for login shells
~/.bashrc User-specific Subshell Personal settings for subshells
~/.bash_history User-specific Always Command history storage

üë§ User Types

4. Ownership and Permissions

üîê Permission Types

Permission Files Directories Octal Value
Read (r) Display contents, copy file List directory contents with ls 4
Write (w) Modify file contents Add/delete files (with execute) 2
Execute (x) Execute file/script Use cd, run ls -l (with read) 1
Ownership Changes:
chown user file # Change user owner
chown user:group file # Change user and group
chown :group file # Change group only
chown -R user dir # Recursive ownership
Permission Changes (Symbolic):
chmod u+x file # Add execute for user
chmod g-w file # Remove write for group
chmod o=r file # Set other to read only
chmod a=rw file # Set all to read/write
Permission Changes (Octal):
chmod 755 file # rwxr-xr-x
chmod 644 file # rw-r--r--
chmod 777 file # rwxrwxrwx
chmod 600 file # rw-------

üéØ Permission Determination Flow

  1. If UID matches file owner's UID ‚Üí Use Owner permissions
  2. Else if GID matches file's group GID ‚Üí Use Group permissions
  3. Else ‚Üí Use Other permissions

3. Pipe Line (|)

üîó Pipeline Concept

Sends the output of one command as the input to another.

Syntax: Command1 | Command2

Pipeline Examples:

ls | wc -l # Count files
Tee Command:

Day 5: Inodes, Archiving, Compression, Package Management, Search

1. Inode Table

üóÇÔ∏è Inode Concept

Linux stores administrative data about files in inodes (index nodes). Within each filesystem, an inode table maps used inodes to specific files.

üìã Inode Contents

  • File type
  • File permissions
  • Number of links (hard links)
  • File owner's user ID
  • Group owner's GID
  • Last changed time
  • Last accessed time
  • Location of file on media

NOT included: File name or file content

Viewing Inodes:
ls -i /etc/passwd # View inode number of file
ls -id /etc # View inode number of directory

2. Hard Links and Symbolic Links

üîó Hard Links

When you create a file, its name is essentially a hard link. Multiple hard links can be created to a file, allowing access from different locations.

Restrictions:

  • Must exist on the same device (partition, logical volume)
  • Cannot create hard links to directories
  • When last name is removed, access to file's data is also removed
Hard Links:
ln source-file target-file # Create hard link
ln /home/user/file hardlink # Example hard link

üîó Symbolic Links (Soft Links)

A new entry is made to the inode table for the link. The content of this entry is the path to the original file.

Advantages:

  • Can link to files on other devices
  • Can link to directories

Disadvantage: When original file is removed, symbolic link becomes invalid

Symbolic Links:
ln -s source-file target # Create symbolic link
ln -s testfile softlink # Example symbolic link

üìÅ File Operations and Inodes

  • cp command: Allocates new inode number, creates new inode table entry
  • mv command (same filesystem): Creates new directory entry with new name but keeps same inode number

3. Archiving (tar)

üì¶ Archive Purpose

To safeguard files and directories by creating a copy (archive) on a removable medium. Archived copies can be used to retrieve lost, deleted, or damaged files.

tar (Tape Archiver): Designed to stream files to backup tapes

Creating Archives:
tar -cvf archive.tar file1 file2 # Create archive
c: Create new tar file
v: Verbose mode (show files)
f: Specify archive filename
Viewing Archives:
tar -tvf archive.tar # List contents
t: List table of contents
Extracting Archives:
tar -xvf archive.tar # Extract files
tar -xvf archive.tar -C /tmp # Extract to specific directory
x: Extract files from archive
Updating Archives:
tar -rvf archive.tar newfile # Append files
tar -uvf archive.tar /home # Update (newer files only)
r: Append files to archive
u: Update archive with newer files

4. Compression

üóúÔ∏è Compression Purpose

To make files take less disk space by removing redundancy. If there's no redundancy, compression won't yield much gain.

Compression Tools:
gzip file.tar # Compress with gzip ‚Üí file.tar.gz
bzip2 file.tar # Compress with bzip2 ‚Üí file.tar.bz2
zip archive.zip files # Create zip archive
Decompression Tools:
gunzip file.tar.gz # Decompress gzip
bunzip2 file.tar.bz2 # Decompress bzip2
zcat file.gz # View compressed content
Combined Archive & Compression:
tar -czvf backup.tar.gz /etc # Create and gzip compress
tar -cjvf backup.tar.bz2 /etc # Create and bzip2 compress
tar -xzvf backup.tar.gz -C /tmp # Extract gzip archive
tar -xjvf backup.tar.bz2 -C /tmp # Extract bzip2 archive
Check File Sizes:
ls -lh file # Human-readable file size
du -sh directory # Disk usage of directory

6. Search (find)

üîç Find Command

The find command searches the live filesystem. It's limited by your own permissions.

Search by Name:
find /path -name "filename" # Find by exact name
find /path -name "*.txt" # Find by pattern
find /path -iname "FILE" # Case insensitive
Search by Size:
find /path -size +100M # Files larger than 100MB
find /path -size -1M # Files smaller than 1MB
find /path -size 500M # Files exactly 500MB
Search by Time:
find /path -atime -7 # Accessed within 7 days
find /path -mtime +30 # Modified more than 30 days ago
find /path -ctime 1 # Changed exactly 1 day ago
Search by Owner & Type:
find /path -user username # Files owned by user
find /path -type f # Files only
find /path -type d # Directories only
find /path -perm 755 # Specific permissions
Advanced Find Examples:
find /home -name "*.log" -size +10M # Large log files
find /tmp -type f -atime +7 -delete # Delete old temp files
find /etc -name "*.conf" -exec grep -l "ssh" {} \; # Config files containing "ssh"

‚ö†Ô∏è Find Command Notes

  • Size values: Use k, M, G for kilobytes, megabytes, gigabytes
  • Time values: Measured in 24-hour periods
  • Use -exec to execute commands on found files
  • Use -delete with caution - it permanently removes files
