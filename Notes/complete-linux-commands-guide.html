👥 6. User & Group Management

🏷️ User Types in Linux

👑 Root User

UID: 0

System administrator with unlimited privileges. Can perform any operation on the system.

# Switch to root
sudo -i
sudo -s
su -
su root

👤 Normal Users

UID: 1000+

Regular users with limited privileges. Each user has their own home directory and settings.

# Regular user operations
whoami
id username
groups username

⚙️ Service Users

UID: 1-999

System accounts for running services and daemons. Usually don't have login shells.

# Examples
# www-data (web server)
# mysql (database)
# mail (mail service)

📋 User Information Files

👥 /etc/passwd - User Database

# View all users
cat /etc/passwd

# Format: user:x:uid:gid:comment:home:shell
# Example:
root:x:0:0:root:/root:/bin/bash
mina:x:1000:1000:Mina User:/home/mina:/bin/bash

Fields: Username, Password (x=encrypted), UID, GID, Comment, Home Directory, Shell

👥 /etc/group - Group Database

# View all groups
cat /etc/group

# Format: group:x:gid:members
# Example:
sudo:x:27:mina,alice
developers:x:1001:john,jane,bob

Fields: Group name, Password, GID, Group members

🔐 Password and Security Files

🛡️ Shadow Files (Root Access Required):

# User passwords (encrypted)
sudo cat /etc/shadow
# Format: user:encrypted_password:last_change:min:max:warn:inactive:expire

# Group passwords
sudo cat /etc/gshadow
# Format: group:encrypted_password:admins:members

➕ Creating Users

🔧 useradd Default Settings:

# View default settings
useradd -D

# Output:
GROUP=100
HOME=/home
INACTIVE=-1
EXPIRE=
SHELL=/bin/sh
SKEL=/etc/skel
CREATE_MAIL_SPOOL=no
# Basic user creation
sudo useradd username

# Create user with specific options
sudo useradd fatma -s /usr/bin/bash -m -d /home/fatma -c "Python Developer"

# Breakdown of options:
# -s /usr/bin/bash → Set shell
# -m → Create home directory
# -d /home/fatma → Specify home directory
# -c "Python Developer" → Add comment
# -u 1004 → Set specific UID
# -g groupname → Set primary group

# Set user password
sudo passwd fatma

# Verify user creation
id fatma
tail -1 /etc/passwd

🏠 User Home Directory Templates

📁 /etc/skel - Skeleton Directory:

Files and directories in /etc/skel are automatically copied to new user home directories.

# Customize new user defaults
sudo mkdir /etc/skel/{Documents,Music,Pictures}
sudo touch /etc/skel/.bashrc_custom

# Now all new users will get these directories

✏️ Modifying Users

Command Description Example
usermod -G group Set secondary groups (replaces) sudo usermod fatma -G sudo,developers
usermod -aG group Add to secondary groups sudo usermod fatma -aG docker
usermod -s shell Change user shell sudo usermod fatma -s /bin/zsh
usermod -d directory Change home directory sudo usermod fatma -d /opt/fatma
usermod -l newname Change username sudo usermod fatma -l fatima

👤 User Modification Examples:

# Add user to sudo group
sudo usermod fatma -aG sudo

# Check user groups
id fatma
# Output: uid=1004(fatma) gid=1004(fatma) groups=1004(fatma),27(sudo)

# Add to multiple groups
sudo usermod fatma -aG sudo,docker,developers

# Verify changes
groups fatma

🗑️ Removing Users

🗂️ Remove User (Keep Home)

sudo userdel username

Removes user account but keeps home directory and files

🗑️ Remove User (Delete All)

sudo userdel -r username

Removes user account, home directory, and all files

👥 Group Management

# Create group
sudo groupadd groupname
sudo groupadd developers -g 1500 # with specific GID

# Create system group (GID < 1000)
sudo groupadd -r system_group

# Modify group
sudo groupmod oldname -n newname # rename group
sudo groupmod groupname -g 1600 # change GID

# Delete group
sudo groupdel groupname

# Add user to group
sudo usermod -aG groupname username

# Remove user from group
sudo gpasswd -d username groupname

🔑 Password Policies

⏰ Password Aging with chage:

# Set password policies
sudo chage username -E -1 # Never expire
sudo chage username -M 90 # Max 90 days
sudo chage username -m 7 # Min 7 days between changes
sudo chage username -W 7 # Warn 7 days before expiry
sudo chage username -I 30 # Account inactive after 30 days

# Set all at once
sudo chage username -E -1 -M 99999 -m 0 -W 7

# View password info
sudo chage username -l

📊 chage Output Example:

Last password change : Jul 21, 2025
Password expires : never
Password inactive : never
Account expires : never
Minimum number of days between password change : 0
Maximum number of days between password change : 99999
Number of days of warning before password expires : 7

🔄 User Switching

Command Environment Working Directory Description
su username Current user's Current Switch user, keep environment
su - username Target user's Target user's home Switch user with login shell
sudo -u user command Current Current Run single command as user
sudo -su user Target user's Target user's home Become user via sudo

🔄 Switching Examples:

# Current context
pwd # /home/mina/projects

# Switch without login shell
su fatma
pwd # /home/mina/projects (stays same)

# Switch with login shell
su - fatma
pwd # /home/fatma (user's home)

# Exit back to previous user
exit

🔐 7. File Permissions & Ownership

👑 Understanding Ownership

🏠 Every file and directory has two owners:

  • User Owner: The user who owns the file
  • Group Owner: The group that owns the file
# Check file ownership
ls -l filename
# -rw-r--r-- 1 mina developers 1024 Jul 23 10:30 document.txt
# │ │
# │ └── Group owner
# └────── User owner

# Check directory ownership
ls -ld directory/
# drwxr-xr-x 2 mina developers 4096 Jul 23 10:30 projects/

🔄 Changing Ownership

👤 Change User Owner

sudo chown newuser file

Changes the user owner of the file

👥 Change Group Owner

sudo chown :newgroup file
sudo chgrp newgroup file

Changes the group owner of the file

👤👥 Change Both

sudo chown user:group file

Changes both user and group owners

🔧 Ownership Examples:

# Change ownership of a single file
sudo chown alice document.txt
sudo chown alice:developers document.txt

# Change ownership recursively
sudo chown -R alice:developers /home/alice/projects/

# Copy ownership from another file
sudo chown --reference=template.txt document.txt

# Change only if current owner matches
sudo chown --from=olduser newuser file.txt

📋 Permission Breakdown

🔍 Reading Permission Strings:

# Example: -rwxr-xr--
# │││││││││
# │││├┼┼┼┼── Others: r-- (read only)
# │││└┼┼┼─── Group: r-x (read, execute)
# ││└─┼┼┼──── Owner: rwx (read, write, execute)
# │└──┼┼┼───── File type: - (regular file)
# └───┴┴┴────── [d=directory, l=link, etc.]

# Permission calculations:
# rwx = 4+2+1 = 7
# r-x = 4+0+1 = 5
# r-- = 4+0+0 = 4
# Result: 754

📁 File vs Directory Permissions

Permission Files Directories
Read (r) View file contents List directory contents (ls)
Write (w) Modify file contents Create/delete files in directory
Execute (x) Run file as program Access directory (cd into it)
⚠️ Directory Permission Notes:
  • Need x permission to cd into a directory
  • Need r permission to list directory contents
  • Need w permission to create/delete files (even if you own the files)
  • Directory permissions override file permissions for access

🔧 Changing Permissions

🔢 Numeric Method

# Common permission combinations
chmod 755 file # rwxr-xr-x
chmod 644 file # rw-r--r--
chmod 600 file # rw-------
chmod 777 file # rwxrwxrwx (dangerous!)

# Recursive changes
chmod -R 755 directory/

🔤 Symbolic Method

# Add permissions
chmod u+x file # Add execute for user
chmod g+w file # Add write for group
chmod o+r file # Add read for others
chmod a+x file # Add execute for all

# Remove permissions
chmod u-w file # Remove write for user
chmod g-x file # Remove execute for group

# Set exact permissions
chmod u=rwx,g=rx,o=r file

📊 Common Permission Patterns

Permissions Numeric Use Case Example
-rw------- 600 Private files SSH keys, passwords
-rw-r--r-- 644 Public readable files Documents, config files
-rw-rw-r-- 664 Group writable files Shared project files
-rwxr-xr-x 755 Executable files Scripts, programs
drwxr-xr-x 755 Public directories Home directories
drwxrwx--- 770 Group shared directories Project folders

🎭 Real-World Permission Examples

🏢 Organizational Permission Structure:

# Library organization example:
# Staff: rwx (full access)
# Students: r-x (read and access)
# Guests: --x (access only)

# Set up project directory
sudo mkdir /projects/webapp
sudo chown :developers /projects/webapp
sudo chmod 775 /projects/webapp

# Staff can read/write/execute
# Students can read/execute
# Others can only execute (access)
chmod 754 /projects/webapp/script.sh

🔐 Special Permissions

🎯 Setuid (SUID)

chmod u+s file
chmod 4755 file

File executes with owner's privileges

👥 Setgid (SGID)

chmod g+s directory
chmod 2755 directory

Files inherit directory's group

🔒 Sticky Bit

chmod +t directory
chmod 1755 directory

Only owner can delete files

🌟 Special Permission Examples:

  • SUID: /usr/bin/passwd - Users can change passwords
  • SGID: /var/mail - Mail files get correct group
  • Sticky: /tmp - Users can't delete others' files

🐚 8. Shell Environment & Variables

🐚 Understanding Shells

# List all available shells
cat /etc/shells

# Output example:
/bin/sh
/bin/bash
/usr/bin/bash
/bin/rbash
/usr/bin/rbash
/usr/bin/sh
/bin/dash
/usr/bin/dash
/usr/bin/zsh
/usr/bin/fish

🔍 Current Shell Information

🐚 Shell Detection

# Current shell path
echo $SHELL
# /usr/bin/bash

# Current shell name
echo $0
# bash

# Shell process info
ps $$

📊 Variable Types

  • $SHELL: Environment variable
  • $0: Predefined variable
  • $$: Current process ID
  • $?: Last command exit status

📝 Variable Types and Usage

🏠 Local Variables

# Set local variable
x=5
name="John Doe"

# Access variables
echo $x # 5
echo $name # John Doe
echo ${name}Ahmed # John DoeAhmed

🌍 Environment Variables

# Common environment variables
echo $HOME # /home/username
echo $USER # username
echo $PWD # current directory
echo $PATH # executable paths
echo $HOSTNAME # computer name

📋 Predefined Variables

# Special shell variables
echo $0 # Shell name/script name
echo $? # Exit status (0=success)
echo $$ # Current process ID
echo $1 $2 # Script arguments

🔍 Viewing Environment Variables

# View all environment variables
env
printenv

# Filter specific variables
env | grep HOME
env | grep USER

# Search for patterns
env | grep -i path # Case-insensitive search

# Show specific variable
printenv HOME
printenv PATH

🎨 Customizing the Prompt

🎭 PS1 - Primary Prompt:

Code Description Example
\u Username mina
\h Hostname server01
\w Current directory ~/Documents
\W Current directory (basename) Documents
\t Time (HH:MM:SS) 14:30:25
\d Date Tue Jul 23
\$ $ for user, # for root $

🎨 Custom Prompt Examples:

# Default format
PS1="\u@\h:\w\$ "
# Output: mina@server:~/Documents$

# With time
PS1="\u:\t@\h:\w\$ "
# Output: mina:16:15:11@server:~/Documents$

# Colorful prompt
PS1="\[\033[01;32m\]\u@\h\[\033[00m\]:\[\033[01;34m\]\w\[\033[00m\]\$ "

# Simple prompt
PS1="[\u@\h \W]\$ "
# Output: [mina@server Documents]$

🔄 PS2 - Secondary Prompt

➡️ PS2 appears when commands span multiple lines:

# Default PS2 is "> "
echo "Hello world \
> this is a long line"

# Customize PS2
PS2=">>> "

echo "Hello \
>>> world"

# Multi-line string
echo "Line 1
>>> Line 2
>>> Line 3"

🛤️ PATH Variable

🔍 Understanding PATH:

# View current PATH
echo $PATH
# /usr/local/bin:/usr/bin:/bin:/usr/local/games:/usr/games

# PATH determines where shell looks for commands
which ls # /usr/bin/ls
which cat # /usr/bin/cat
which python3 # /usr/bin/python3
# Add directory to PATH temporarily
export PATH=$PATH:~/bin
export PATH=$PATH:~/Music/scripts

# Add to beginning of PATH (higher priority)
export PATH=~/bin:$PATH

# Create custom script
mkdir ~/bin
echo '#!/bin/bash' > ~/bin/myscript
echo 'echo "Hello from custom script!"' >> ~/bin/myscript
chmod +x ~/bin/myscript

# Add to PATH and test
export PATH=$PATH:~/bin
myscript # Hello from custom script!

📝 ~/.bashrc Customization:

# Add to ~/.bashrc

# Custom aliases
alias ll='ls -la'

alias l='ls -CF'
alias c='clear'
alias ..='cd ..'
alias ...='cd ../..'

mkcd() {

}

🔄 Login vs Non-Login Shell Examples

🚪 Shell Loading Behavior:

# Login shell (su - username)
su - ali
# Loads: /etc/profile → ~/.profile → /etc/bash.bashrc → ~/.bashrc
pwd # /home/ali (user's home)

# Non-login shell (su username)
su ali
# Loads: /etc/bash.bashrc → ~/.bashrc
pwd # /current/directory (stays same)

# Exit back to previous user
exit

🔧 Variable Best Practices

⚠️ Variable Guidelines:
  • Use export to make variables available to child processes
  • Quote variables with spaces: name="John Doe"
  • Use ${variable} syntax to avoid ambiguity
  • Make permanent changes in ~/.bashrc or ~/.profile
  • Reload configuration with source ~/.bashrc

🔄 Reloading Configuration:

# After editing ~/.bashrc, reload it:
source ~/.bashrc
# or
. ~/.bashrc

# Check if changes took effect
echo $CUSTOM_VARIABLE
alias # List all aliases
