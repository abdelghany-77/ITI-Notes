Lesson 1: Introduction to PHP

What is PHP and why it's important for web development

1.1 What is PHP?

PHP (Hypertext Preprocessor) is a popular open-source server-side scripting language designed specifically for web development. It can be embedded directly into HTML and is used to create dynamic web pages.

Key Features:

  • Server-side scripting language
  • Open-source and free
  • Cross-platform compatibility
  • Large community and ecosystem
  • Easy integration with databases
  • Rich built-in functions

1.2 PHP vs Other Languages

PHP

Server-side, web-focused, easy to learn, vast hosting support

JavaScript (Node.js)

Full-stack capable, event-driven, single-threaded with async

Python

General-purpose, clean syntax, strong in data science and AI

Java

Enterprise-grade, strongly typed, excellent for large applications

1.3 Setting Up PHP Environment

XAMPP Installation

XAMPP is a free and open-source cross-platform web server solution stack package.

  1. Download XAMPP from apachefriends.org
  2. Run the installer as administrator
  3. Start Apache and MySQL services
  4. Place PHP files in htdocs folder
  5. Access via localhost in browser

Lesson 3: Variables & Data Types

Understanding PHP variables, data types, and type juggling

3.1 PHP Variables

Variables in PHP are represented by a dollar sign ($) followed by the variable name. Variable names are case-sensitive and must start with a letter or underscore.

Variable Declaration & Assignment

Lesson 6: Functions

Defining, calling, and working with functions in PHP

6.1 Defining Functions

Functions in PHP are defined using the function keyword. Function names are not case-sensitive, but cannot start with a digit.

Basic Function Definition
<?php
function getSum($x, $y) {
    $z = $x + $y;
    echo 'Sum of x and y is ' . $z . "<br>";
}
// Function names are case-insensitive
getSum(55, 55);  // Works
getsum(5, 10);   // Also works
?>

Deleting Functions

Using unset() to Delete Functions
<?php
function myFunction() {
    echo "Hello World!";
}
myFunction();  // Works
// Delete the function
unset($myFunction);
myFunction();  // Fatal error: function not found
?>

6.2 Function Parameters & Return

Optional Parameters

Functions with Default Values
<?php
function add($x, $y = 1) {
    $z = $x + $y;
    echo 'Sum of x and y is ' . $z . "<br>";
}
add(6);      // Uses default y=1, outputs: Sum of x and y is 7
add(6, 7);   // Uses provided y=7, outputs: Sum of x and y is 13
?>

Variable Length Arguments

Using ... Token for Variable Arguments
<?php
function getSum($op1, ...$op2) {
    foreach ($op2 as $op) {
        $op1 += $op;
    }
    return $op1;
}
echo getSum(1, 2, 3, 6);  // Outputs: 12
?>

Return Keyword

Using return to Exit Function
<?php
function mul($x, $y) {
    return $x * $y;
    echo 'This line will never be executed';
}
$result = mul(5, 6);
echo $result . "<br>";  // Outputs: 30
?>

6.3 Call by Value vs Call by Reference

Call by Value

Original Value Unchanged
<?php
function incrementFun($value, $amount = 1) {
    $value = $value + $amount;
}
$value = 10;
incrementFun($value);
var_dump($value);  // Outputs: int(10) - unchanged
?>

Call by Reference

Original Value Modified
<?php
function incrementFun(&$value, $amount = 1) {
    $value = $value + $amount;
}
$value = 10;
incrementFun($value);
var_dump($value);  // Outputs: int(11) - modified
?>

Lesson 7: Closures

Anonymous functions and closures in PHP

7.1 What are Closures?

A closure is the PHP equivalent of an anonymous function - a function that does not have a name. Closures are objects of the Closure class.

Basic Closure Example
<?php
$greet = function($name) {
    printf("Hello %s\r\n", $name);
};  // Semicolon is required
$greet('World');  // Outputs: Hello World
var_dump(is_callable($greet));  // bool(true)
?>

7.2 Closures and External Variables

Use the use keyword to access variables from the outer scope.

Using External Variables
<?php
$quantity = 1;
$calculator = function($number) use($quantity) {
    return $number + $quantity;
};
var_dump($calculator(7));  // int(8)
?>

7.3 Binding Closures to Classes

Using bindTo()

Binding to Public Properties
<?php
$myClosure = function() {
    echo $this->property . "<br>";
};
class MyClass {
    public $property;
    function __construct($propertyValue) {
        $this->property = $propertyValue;
    }
}
$objClass = new MyClass("Hello from closure");
$myBoundClosure = $myClosure->bindTo($objClass);
$myBoundClosure();  // Outputs: Hello from closure
?>
Binding to Private Properties
<?php
$myClosure = function() {
    echo $this->property . "<br>";
};
class MyClass {
    private $property;
    function __construct($propertyValue) {
        $this->property = $propertyValue;
    }
}
$objClass = new MyClass("Hello from closure");
$myBoundClosure = $myClosure->bindTo($objClass, MyClass::class);
$myBoundClosure();  // Outputs: Hello from closure
?>

Using call() Method (PHP 7+)

Direct Closure Call
<?php
$myClosure = function() {
    echo $this->property . "<br>";
};
class MyClass {
    private $property;
    function __construct($propertyValue) {
        $this->property = $propertyValue;
    }
}
$objClass = new MyClass("Hello from closure using call");
$myClosure->call($objClass);  // Outputs: Hello from closure using call
?>

7.4 Closures Inside Classes

Defining Closure in Class Method
<?php
class NewClass {
    private $prop;
    function __construct($propertyValue) {
        $this->prop = $propertyValue;
    }
    function display() {
        return function() {
            echo $this->prop . "<br>";
        };
    }
}
$obj = new NewClass("Hello World");
$display = $obj->display();
$display();  // Outputs: Hello World
?>

Lesson 8: Object-Oriented Programming

Classes, objects, inheritance, and OOP principles in PHP

8.1 OOP Principles

Encapsulation

Binds data and functions together, keeping them safe from outside interference

Inheritance

Allows classes to be arranged in hierarchy representing "is-a-type-of" relationships

Polymorphism

Ability to take more than one form - same interface, different implementations

Abstraction

Hides complexity by showing only necessary details to the user

8.2 Classes and Objects

Classes are blueprints for objects. Objects are instances of classes. Classes can have properties (attributes) and methods (functions).

Basic Class Definition
<?php
class Person {
    public $name;  // Public property
    function sayHello() {
        echo "Hello " . $this->name;
    }
}
$p = new Person();
$p->name = "Noha";
$p->sayHello();  // Outputs: Hello Noha
?>

8.3 Access Modifiers

Public

Accessible from inside and outside the class (default for methods)

Private

Accessible only from inside the class, not inherited

Protected

Accessible from inside the class and subclasses

Static

Belongs to the class, not instances

Access Modifiers Example
<?php
class Person {
    public $name;
    private $first_name;
    protected $last_name;
    private function sayHello() {
        echo "Hello " . $this->first_name;
    }
}
?>

8.4 Constructors and Destructors

Constructor

__construct() Method
<?php
class Person {
    private $first_name;
    private $last_name;
    function __construct($first_name, $last_name) {
        $this->first_name = $first_name;
        $this->last_name = $last_name;
    }
}
$p = new Person("Noha", "Shehab");
?>

Destructor

__destruct() Method
<?php
class Person {
    function __destruct() {
        echo "<br> Destructing object";
    }
}
$p = new Person("Noha", "Shehab");
// Object will be destroyed when script ends
unset($p);  // Explicit destruction
?>

8.5 Magic Methods (__get & __set)

Dynamic Getters and Setters
<?php
class Person {
    function __set($name, $value) {
        $this->$name = $value;
    }
    function __get($name) {
        return $this->$name;
    }
}
$p = new Person();
$p->__set("track", "opensource");
echo $p->__get("track");  // Outputs: opensource
?>

8.6 Static Methods and Properties

Static Members
<?php
class Math {
    const PI = 3.14159;
    static $mul = 1;
    static function squared($input) {
        return $input * $input;
    }
    function testSelf() {
        echo self::PI;
    }
}
echo Math::squared(8);  // Outputs: 64
?>

$this refers to the current object instance
self refers to the current class (use self::$member for static members)

Lesson 9: Inheritance

Creating class hierarchies and extending functionality

9.1 Basic Inheritance

PHP supports single inheritance using the extends keyword. Child classes inherit properties and methods from parent classes.

Inheritance Example
<?php
class Person {
    public $name;
    public function callingPerson() {
        echo "I am a person";
    }
}
class Student extends Person {
    public $level;
    public function callingStudent() {
        echo "I am a student";
    }
}
$std = new Student();
$std->name = "Omar";
$std->level = 1;
$std->callingPerson();   // Inherited method
$std->callingStudent();  // Own method
?>

9.2 Parent Constructor

Calling Parent Constructor
<?php
class Person {
    public $name;
    public function __construct() {
        echo "Creating a person ";
    }
}
class Student extends Person {
    public $level;
    public function __construct() {
        parent::__construct();  // Call parent constructor
        echo "Creating a student";
    }
}
$std = new Student();  // Outputs: Creating a person Creating a student
?>

9.3 Method Overriding

Overriding Parent Methods
<?php
class Person {
    public $name;
    public function say_hello() {
        echo "Hello person";
    }
}
class Student extends Person {
    public $level;
    public function say_hello() {
        echo "Hello student";  // Override parent method
    }
}
$person = new Person();
$student = new Student();
$person->say_hello();   // Outputs: Hello person
$student->say_hello();  // Outputs: Hello student
?>

9.4 Final Keyword

final methods cannot be overridden
final classes cannot be extended

Final Methods and Classes
<?php
class Machine {
    public $brand;
    function sayHello() {
        echo "Hello I am a machine";
    }
}
class Transportation extends Machine {
    public $type;
    final function canMove() {
        echo "I can move";
    }
}
final class Car extends Transportation {
    public $model;
    function sayHello() {
        parent::sayHello();
        echo " Called from the Car class";
    }
    // This would cause error:
    // function canMove() { }  // Cannot override final method
}
// This would cause error:
// class SportsCar extends Car { }  // Cannot extend final class
?>

Lesson 10: Abstract Classes & Interfaces

Abstract classes, interfaces, and polymorphism

10.1 Abstract Classes

Abstract classes contain abstract methods that must be implemented by child classes. Abstract classes cannot be instantiated directly.

Abstract Class Example
<?php
abstract class Base {
    function __construct() {
        echo "<br> <b> This is abstract class constructor ";
    }
    // Abstract method - no body
    abstract function printdata();
}
class Derived extends Base {
    function __construct() {
        echo "<br> <b> Derived class constructor";
        parent::__construct();
    }
    function printdata() {
        echo "<br> <b> Derived class printdata function";
    }
}
$b1 = new Derived();  // Cannot instantiate Base directly
$b1->printdata();
?>

10.2 Interfaces

Interfaces specify methods that classes must implement without providing the implementation. Classes can implement multiple interfaces.

Interface Example
<?php
interface Transportation {
    public function setModel($model);
    public function setYear($year);
}
class Car implements Transportation {
    public $model;
    public $year;
    public function setModel($model) {
        $this->model = $model;
    }
    public function setYear($year) {
        $this->year = $year;
    }
}
$car = new Car();
$car->setModel("Toyota");
$car->setYear(2023);
?>

10.3 Interfaces vs Abstract Classes

Feature Abstract Classes Interfaces
Properties Can have properties Cannot have properties
Method Visibility Public or protected Only public
Method Implementation Can have implemented methods All methods are abstract
Multiple Inheritance Not supported Supported

10.4 Anonymous Classes

Anonymous Class Implementation
<?php
interface DisplayMsg {
    public function printMsg(string $msg);
}
class Application {
    private $displayer;
    public function getDisplayer(): DisplayMsg {
        return $this->displayer;
    }
    public function setPrinter(DisplayMsg $dismsg) {
        $this->displayer = $dismsg;
    }
}
$app = new Application();
$app->setPrinter(new class implements DisplayMsg {
    public function printMsg(string $msg) {
        echo($msg) . "<br>";
    }
});
$app->getDisplayer()->printMsg("My first Log Message using anonymous class");
?>

Lesson 11: Advanced OOP Concepts

Reflection, polymorphism, cloning, and traits

11.1 Reflection Classes

Reflection allows a program to inspect and modify its own structure at runtime. This is also called "introspection".

Using Reflection
<?php
class OpenSource {
    private $instructor;
    protected $sub_tracks;
    public $list_of_courses;
    const PI = 3.1415;
    public function __construct() {
        $this->instructor = "Noha";
        $this->sub_tracks = "Application";
        $this->list_of_courses = ["Python", "PHP", "Scala", "Laravel", "Admin"];
    }
    public function getInstructor() {
        return $this->instructor;
    }
    public function setInstructor($instructor) {
        $this->instructor = $instructor;
    }
    private function getsub_tracks() {
        return $this->sub_tracks;
    }
}
$reflection_class = new ReflectionClass("OpenSource");
foreach ($reflection_class->getMethods() as $method) {
    echo $method->getName() . "<br>";
}
?>

When to use Reflection:

  • Unit testing applications
  • Getting information about unknown classes
  • Dynamic code analysis

11.2 Polymorphism

Polymorphism allows objects of different classes to respond differently to the same message. Achieved through interfaces or abstract classes.

Method Overriding (Dynamic Binding)
<?php
interface Animals {
    public function makeNoise();
}
class Cat implements Animals {
    public function makeNoise() {
        $this->meow();
    }
    function meow() {
        echo '<br> meow';
    }
}
class Dog implements Animals {
    public function makeNoise() {
        $this->bark();
    }
    function bark() {
        echo '<br> bark';
    }
}
$cat = new Cat();
$dog = new Dog();
$cat->makeNoise();  // Outputs: meow
$dog->makeNoise();  // Outputs: bark
?>

11.3 Dynamic Overloading

Using __call and __callStatic
<?php
class phpClass {
    public function __call($name, $arguments) {
        echo "Calling object method '$name' " . implode(', ', $arguments) . "<br>";
    }
    public static function __callStatic($name, $arguments) {
        echo "Calling static method '$name' " . implode(', ', $arguments) . "<br>";
    }
}
$obj = new phpClass();
$obj->runTest('in object context');
phpClass::runTest('in static context');
?>

11.4 Object Cloning

Object cloning creates a copy of an object using the clone keyword. PHP does shallow copying by default. Use __clone() for deep copying.

Basic Object Cloning
<?php
class Product {
    public $name;
    public $category;
}
$objProduct = new Product();
$objProduct->name = "Apple";
$objProduct->category = "Fruit";
// Clone the object
$objCloned = clone $objProduct;
$objCloned->name = "Orange";
print_r($objProduct);  // Original: Apple, Fruit
print_r($objCloned);   // Clone: Orange, Fruit
?>

Lesson 12: Traits

Code reuse through traits in PHP

12.1 What are Traits?

Traits are a mechanism for code reuse in single inheritance languages like PHP. They allow you to "copy and paste" functionality into classes. Multiple traits can be used in a single class.

Basic Trait Usage
<?php
trait Hello {
    public function sayHello() {
        echo ' <br> Hello ';
    }
}
trait World {
    public function sayWorld() {
        echo 'World';
    }
}
class MyHelloWorld {
    use Hello, World;
    public function sayExclamationMark() {
        echo '!';
    }
}
$o = new MyHelloWorld();
$o->sayHello();           // Outputs: Hello
$o->sayWorld();           // Outputs: World
$o->sayExclamationMark(); // Outputs: !
?>

12.2 Include and Require

include() and require() load external files.
include_once() and require_once() ensure files are loaded only once.

File Inclusion
<?php
// Include a file (warning if not found)
include('functions.php');
// Require a file (fatal error if not found)
require('config.php');
// Include only once
include_once('database.php');
require_once('utilities.php');
?>

Lesson 14: Error Handling

Managing errors and exceptions in PHP applications

14.1 Error Types

Critical errors that stop script execution

Non-critical errors that allow continuation

14.2 Try-Catch Blocks

Exception Handling
try {
    // Code that might throw an exception
    $result = 10 / 0;
    echo "Result: " . $result;
} catch (Exception $e) {
    // Handle the exception
    echo "Error: " . $e->getMessage();
} finally {
    // Always executed
    echo "Finally block executed";
}

14.3 Custom Exceptions

Creating Custom Exception Classes
class CustomException extends Exception {
    public function errorMessage() {
        return 'Custom Error: ' . $this->getMessage();
    }
}
try {
    throw new CustomException("Something went wrong");
} catch (CustomException $e) {
    echo $e->errorMessage();
}

14.4 Error Reporting

Configuring Error Display
// Display all errors during development
ini_set('display_errors', 1);
ini_set('display_startup_errors', 1);
error_reporting(E_ALL);
// Log errors to file in production
ini_set('log_errors', 1);
ini_set('error_log', '/path/to/error.log');
// Custom error handler
function customError($errno, $errstr) {
    echo "<b>Custom Error:</b> [$errno] $errstr<br>";
}
set_error_handler("customError");

Lesson 16: Sessions & Cookies

Managing user state with sessions and cookies

16.1 Cookies

Cookies are small text files stored on the client-side to maintain state between HTTP requests. They're sent with every request to the server.

Setting and Getting Cookies
// Set a cookie (expires in 1 hour)
setcookie("username", "JohnDoe", time() + 3600, "/");
// Set cookie with more options
setcookie("user_id", "123", [
    'expires' => time() + 86400,  // 24 hours
    'path' => '/',
    'domain' => '',
    'secure' => true,
    'httponly' => true,
    'samesite' => 'Strict'
]);
// Get cookie value
if (isset($_COOKIE['username'])) {
    $username = $_COOKIE['username'];
    echo "Welcome back, " . $username;
}
// Delete cookie
setcookie("username", "", time() - 3600);

16.2 Sessions

Sessions store user data on the server-side. A session ID is stored in a cookie on the client-side to identify the user across multiple requests.

Working with Sessions
// Start session
session_start();
// Store session data

16.3 Session Security

Security Best Practices:

  • Always call session_start() at the beginning of your scripts
  • Use session_regenerate_id() after login to prevent session fixation
  • Set session.cookie_httponly to prevent XSS attacks
  • Use session.cookie_secure for HTTPS-only sites
  • Implement session timeout mechanisms
  • Store sensitive data in sessions, not cookies
Secure Session Configuration
// Configure secure session settings
ini_set('session.cookie_httponly', 1);
ini_set('session.cookie_secure', 1);
ini_set('session.use_only_cookies', 1);
session_start();
// Regenerate session ID after login
if (isset($_SESSION['logged_in'])) {
    session_regenerate_id(true);
}
// Session timeout (30 minutes)
$timeout = 1800; // 30 minutes in seconds
if (isset($_SESSION['last_activity']) &&
    (time() - $_SESSION['last_activity']) > $timeout) {
    session_destroy();
    header('Location: login.php');
}
$_SESSION['last_activity'] = time();

Lesson 18: Security Best Practices

Protecting your PHP applications from common vulnerabilities

18.1 SQL Injection Prevention

SQL Injection occurs when user input is not properly sanitized and gets executed as SQL code.

Vulnerable Code (Don't Do This)
// VULNERABLE - Never do this!
$username = $_POST['username'];
$sql = "SELECT * FROM users WHERE username = '$username'";
$result = mysqli_query($conn, $sql);
Secure Code (Use Prepared Statements)
// SECURE - Use prepared statements
$stmt = $conn->prepare("SELECT * FROM users WHERE username = ?");
$stmt->bind_param("s", $_POST['username']);
$stmt->execute();
$result = $stmt->get_result();

18.3 CSRF (Cross-Site Request Forgery) Protection

CSRF Token Implementation
session_start();

if (!isset($_SESSION['csrf_token'])) {
    $_SESSION['csrf_token'] = bin2hex(random_bytes(32));
}
// In your HTML form

// Verify CSRF token on form submission

    if (!isset($_POST['csrf_token']) ||
        $_POST['csrf_token'] !== $_SESSION['csrf_token']) {
        die('CSRF token validation failed');
    }
    // Process the form
}

function loginUser($email, $password) {
    global $conn;
